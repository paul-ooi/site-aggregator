Less than a year ago, a client contacted our support line with an unusual situationâ€¦users were "disappearing from the platform." Multiple inquiries had come from colleagues who had been set up in DuBot but had now disappeared and no longer had access.Â 

After some sleuthing, DubBot engineers determined that it was "possible that an admin inadvertently removed users, but we (DubBot) do not currently track those events."

In January 2024, DubBot launched the Account History feature, which tracks when users are added or removed from their DubBot account, including who took the action. The feature also tracks ignoring and unignoring issues.

![Previous version of the Account History page with a list of the latest changes made. A pink box highlights the action of an admin adding a person to the account.](../../images/blog-content/2025/acct-history-remove-person.png)

Changes you can track for sites include creating / deleting sites, enabling / disabling sites for checking (crawling), enabling PDF checking, and page set membershipâ€”any actions that can be applied to your sites can be tracked.

DubBot instantly records any changes to your site. In the Changes column, you will see an icon in the shape of a human eye.

![](../../images/blog-content/2025/site-changes.png)

Clicking the "eye" icon provides a pop-up window with the details of the change, which include the message listed on the Site changes panel, the user who made the change, the date the change was made, and what the change was. In the screenshot below, the change was "Crawler Configuration - Obey Robots Txt."

![](../../images/blog-content/2025/acct-history-popup.png)

DubBot has rapidly expanded the Account History feature from four to eight trackable items, making it even easier to maintain a clear audit trail for regulatory compliance and troubleshooting purposes.

The data in the Account History feature is also valuable for maintaining user accountability, reporting user activity, and providing error recovery and rollback support.
